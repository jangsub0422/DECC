DECC Pipeline 사용자 안내서

1. 개요

이 프로젝트는 자연어 요청을 여러 개의 plasmid JSON으로 분해하고,
각 plasmid를 Python 모듈로 생성한 뒤,
마지막에 integration 단계로 연결하는 로컬 AI 보조 파이프라인입니다.

면책사항:

이 프로젝트는 AI 보조 개발 워크플로우를 사용합니다.
분해, 코드 생성, 검증, integration 과정에 LLM 또는 로컬 코딩 모델이 관여합니다.
따라서 생성 결과에는 오류, 누락, 비효율, 보안 문제, 잘못된 가정이 포함될 수 있습니다.
실사용 전에는 반드시 사용자가 직접 검토하고 검증해야 합니다.
개발자 또는 배포자는 생성 결과의 정확성, 완전성, 적합성을 보증하지 않습니다.
실서비스, 민감 데이터, 법률, 의료, 재무, 보안 관련 목적에 사용하기 전에는
추가 검토를 강하게 권장합니다.

기본 흐름:

1) architect.py 또는 web_app.py
2) plasmids_batch/runs/<run_name>/
3) run_batch.py
4) outputs/runs/<run_name>/
5) integration.py
6) final_app.py


2. 빠른 실행 파일

운영체제별 런처 파일:

- Windows: start_decc_windows.bat
- Linux: start_decc_linux.sh

이 파일들은 Streamlit wrapper를 실행합니다.


3. 준비 사항

- Python 설치
- Ollama 설치 및 실행
- 필요한 경우에만 API 키 준비
  - Gemini: GOOGLE_API_KEY 또는 GEMINI_API_KEY
  - OpenAI: OPENAI_API_KEY
  - Claude: ANTHROPIC_API_KEY

의존성 설치:

pip install -r requirements.txt

참고:

- `requirements.txt`는 Python 패키지만 설치합니다.
- Ollama는 별도 설치가 필요한 로컬 서비스입니다.
- Gemini, OpenAI, Claude 같은 상용 API 경로도 지원하지만, 사용량 비용이 발생할 수 있어서 기본 권장 경로는 아닙니다.
- 현재 알파 단계에서는 상용 API 경로가 architect 분해 단계까지만 연결되어 있고,
  모듈 코드 생성 단계는 아직 로컬 Ollama 기반 경로를 사용합니다.


4. 가장 쉬운 사용 방법: web_app.py

Streamlit wrapper는 멀티라인 입력과 run 관리 UI를 제공합니다.

직접 실행:

streamlit run web_app.py

또는 운영체제에 맞는 런처 파일을 실행합니다.

주요 탭:

- Architect
  자연어 요구사항 입력
  Host 선택
  Model 선택
  Run name / project name 입력
  plasmid JSON 생성

- Current Plasmids
  현재 active run의 plasmid JSON 확인

- Runs
  이전 run 목록 확인
  active run 전환
  선택한 run의 plasmid / outputs / final_app snapshot 확인

- Pipeline
  run_batch 실행
  integration 실행
  현재 active output run 확인

- Maintenance
  legacy root 파일 정리
  __pycache__ 정리


5. CLI 실행 방법

plasmid를 생성한 뒤 아래 순서로 실행합니다.

python run_batch.py
python integration.py
python final_app.py

환경에 따라 python 대신 다른 Python 실행 명령을 사용해도 됩니다.


6. run 이름과 저장 구조

Run name / project name을 입력하면 timestamp와 결합된 run 이름이 만들어집니다.

예:

csv_to_json_20260504_193000

저장 구조:

- plasmids_batch/runs/<run_name>/plasmid_*.json
- outputs/runs/<run_name>/*.py
- outputs/runs/<run_name>/final_app.py


7. 권장 사용 순서

1) 요구사항 입력
2) Run name 입력
3) plasmid 생성
4) Current Plasmids 검토
5) run_batch 실행
6) integration 실행
7) final_app.py 테스트

9. 현재 알파 버전 한계

- 생성 모듈이 contract를 통과하지 못할 수 있습니다.
- 일부 프로파일은 추가적인 prompt 조정이 필요할 수 있습니다.
- profile 범위는 아직 제한적입니다.
- 결과 품질은 선택한 모델에 크게 영향을 받습니다.
- 이 프로젝트는 실험적 알파 단계이며, 생산 환경용 완성품이 아닙니다.
